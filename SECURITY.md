# Security policy

## Reporting a vulnerability

Please do not open a public issue for a vulnerability in BugBountyScout.
Instead, use GitHub's private vulnerability reporting feature for this
repository. Include affected versions, reproduction steps using synthetic data,
impact, and a suggested mitigation when possible.

Never include live credentials, tokens, customer data, or unauthorized target
data in a report. Redact all sensitive evidence before submission.

## Supported versions

Until the first stable release, security fixes are applied to the latest
version on the default branch.

## Responsible-use boundary

BugBountyScout is intended only for owned assets, labs, and explicitly
authorized bug bounty targets. Reports requesting exploit automation,
authentication bypass, rate-limit evasion, secret validation against live
providers, credential theft, or data exfiltration will not be accepted.
